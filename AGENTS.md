# 项目规范指南

## 全局规则

- 回答、注释、文档、Commit 统一使用中文；代码变量、函数、接口字段使用英文。
- 读写文件默认使用 UTF-8。
- 保持输出精炼，除非用户要求，不解释未变更的模板代码。
- 每次修改前先定位相关文件，避免盲目读大文件。
- 手术式修改，只动当前需求相关区域，不顺手重构无关代码。
- 修改 SQL 时优先在 `docs/init.sql` 底部追加增量 SQL，因为旧版脚本可能已经执行过。
- 用户多次要求“不需要编译”时，不跑正式构建；可跑 `tsc --noEmit` 这类类型检查来验证。
- 每次回答保持 `caveman` 风格：短、硬、信息完整。

## 当前项目

YY论坛系统，三端架构：

- `client/`：用户端 Vue 3 + Vue Router + Axios + Tailwind v4。
- `admin/`：管理端 Vue 3 + Vue Router + Axios + Tailwind v4。
- `server/`：NestJS + TypeORM + MySQL。

常用端口：

- 用户端：`3000`
- 后端：`3001`
- 管理端：`3002`

常用命令：

```powershell
cd server
npm run dev

cd client
npm run dev

cd admin
npm run dev
```

## 数据库

- 数据库：MySQL，库名 `forum`。
- 后端连接配置在 `server/src/app.module.ts`。
- 主要表：`user`、`board`、`post`、`comment`、`post_like`。
- `post_like` 用于避免 MySQL 关键字 `like` 冲突。
- `board.sort_order` 控制用户端频道导航顺序，`推荐` 固定在前端，不入库。
- 用户资料字段：`avatar_url`、`signature`。

## 架构约束

- 严守 Controller / Service / Entity 分层。
- Controller 只处理路由、参数、权限入口；业务逻辑放 Service。
- 前端 API 请求统一放在 `api/` 目录。
- 登录态使用现有 `localStorage` token 方案。
- 用户端和管理端 token key 隔离。
- 不引入新依赖，除非现有栈无法完成需求；新增前必须检查 `package.json`。

## 新用户端设计方向

用户端为“YY论坛图片发现页”：

- 顶部居中搜索框，左侧品牌，右侧轻量操作入口。
- 左侧固定导航：发现、直播、发布、通知、我、更多。
- 主内容区为瀑布流/多列卡片信息流。
- 顶部频道栏第一项固定 `推荐`，后续来自数据库板块。
- 管理员创建板块后，用户端频道栏自动追加对应入口。
- 视觉关键词：白底、圆角胶囊、轻量阴影、大图卡片、头像昵称、点赞数。
- 首页 `/boards` 作为 `推荐` 页，不再只是传统板块宫格。
- 帖子第一版改为“图片帖”，只允许发布图片内容；文字只作为标题/说明/标签，不再做纯文本长帖。
- 帖子详情页采用图片社区详情布局：左侧大图预览，右侧作者信息、标题、正文说明、标签、评论区和底部互动栏。
- 如果帖子有多张图片，左侧区域支持轮播/分页点；第一版先保证图片展示，不做视频。

## 已实现重点

- 板块列表、详情、管理端 CRUD。
- 发帖、帖子详情、编辑、删除。
- 帖子图片上传、删除、详情图片展示。
- 评论发布、删除。
- 点赞 toggle。
- 管理端用户、板块、帖子、评论管理。
- 个人中心：头像、签名、改密码、我的帖子。
- 用户头像在导航、个人中心、帖子详情、评论区展示。

  - `docs/api-client.csv`

## 验证习惯

- 后端类型检查：`cd server && npx tsc --noEmit`
- 前端当前没有稳定类型检查脚本；不要随意引入新工具。
- 若用户明确说“不需要编译”，不要跑 `npm run build`。
- 发现测试或类型检查被现有配置阻塞，要说明阻塞原因，不强行重构配置。
