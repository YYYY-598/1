# 论坛项目设计文档

## 一、项目概述

一个轻量级论坛系统，包含用户端、管理端和后端服务。核心功能围绕板块 → 帖子 → 评论三级结构展开，支持点赞交互。

- **用户角色**：普通用户（注册后默认为 user）、管理员（admin）
- **核心流程**：注册 → 登录 → 浏览板块 → 查看帖子列表 → 查看帖子详情 + 评论 → 发帖 / 评论 / 点赞

## 四、数据库设计

### 4.1 实体关系

```
User 1 ──┬── N Post ── N ── 1 Board
          │
          ├── N Comment ── N ── 1 Post
          │
          └── N Like ── N ── 1 Post
```

### 4.2 User

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE, NOT NULL | 邮箱 |
| password | VARCHAR(255) | NOT NULL | bcrypt 哈希 |
| role | ENUM('user','admin') | NOT NULL, DEFAULT 'user' | 角色 |
| is_banned | BOOLEAN | NOT NULL, DEFAULT FALSE | 是否封禁 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 注册时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 更新时间 |

### 4.3 Board

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| name | VARCHAR(50) | UNIQUE, NOT NULL | 板块名 |
| description | VARCHAR(200) | NOT NULL, DEFAULT '' | 板块描述 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |

### 4.4 Post

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| title | VARCHAR(200) | NOT NULL | 标题 |
| content | TEXT | NOT NULL | 正文 |
| user_id | INT | FK → user.id, NOT NULL | 发帖人 |
| board_id | INT | FK → board.id, NOT NULL | 所属板块 |
| like_count | INT | NOT NULL, DEFAULT 0 | 点赞数（冗余计数） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 发帖时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 编辑时间 |

### 4.5 Comment

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| content | TEXT | NOT NULL | 评论内容 |
| user_id | INT | FK → user.id, NOT NULL | 评论人 |
| post_id | INT | FK → post.id, NOT NULL | 所属帖子 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 评论时间 |

### 4.6 Like

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| user_id | INT | FK → user.id, NOT NULL | 点赞人 |
| post_id | INT | FK → post.id, NOT NULL | 被赞帖子 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 点赞时间 |

> **唯一约束**：`UNIQUE(user_id, post_id)`，防止同一用户对同一帖子重复点赞。
>
> **点赞数同步**：`post.like_count` 是冗余字段，在 `Like` 表增删时同步更新 `post.like_count`，避免高频 COUNT 查询。

## 五、API 设计

### 5.0 通用约定

**基础路径**：`/api`

**认证方式**：登录成功后返回 JWT token，前端存 localStorage，请求时通过 Header 携带：
```
Authorization: Bearer <token>
```

**分页参数**：列表类接口统一使用 query 参数 `?page=1&pageSize=10`，默认 `page=1, pageSize=10`。

**分页响应格式**：
```json
{
  "items": [],
  "total": 100,
  "page": 1,
  "pageSize": 10
}
```

**状态码约定**：
| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 / token 过期 |
| 403 | 无权限（非本人 / 非管理员） |
| 404 | 资源不存在 |
| 409 | 冲突（如重复点赞、用户名已存在） |

---

### 5.1 认证

#### POST /api/auth/register
注册新用户，成功后返回 token。

**Body**：
```json
{
  "username": "zhangsan",
  "email": "zhangsan@example.com",
  "password": "123456"
}
```

**校验**：username 长度 2-50，email 合法格式，password 长度 6-100。

#### POST /api/auth/login
登录，成功后返回 token。

**Body**：
```json
{
  "email": "zhangsan@example.com",
  "password": "123456"
}
```

---

### 5.2 板块

#### GET /api/boards
获取所有板块列表，无需分页（板块数量有限）。

#### GET /api/boards/:id
获取单个板块详情。

---

### 5.3 帖子

#### GET /api/boards/:boardId/posts?page=1&pageSize=10
获取某板块下的帖子列表，分页。返回字段不含正文全文，仅含前 200 字摘要。

**响应示例**：
```json
{
  "items": [
    {
      "id": 1,
      "title": "欢迎来到新手村",
      "summary": "这是正文的前200字...",
      "username": "zhangsan",
      "like_count": 5,
      "comment_count": 3,
      "created_at": "2026-05-02T12:00:00.000Z"
    }
  ],
  "total": 42,
  "page": 1,
  "pageSize": 10
}
```

#### GET /api/posts/:id
获取帖子详情，包含帖子全文、作者信息、评论列表。

#### POST /api/boards/:boardId/posts
发帖，需登录。

**Body**：
```json
{
  "title": "帖子标题",
  "content": "帖子正文..."
}
```

**校验**：title 长度 1-200，content 长度 1-10000。

#### DELETE /api/posts/:id
删帖。只能删自己的帖子，管理员可删任意帖子。

---

### 5.4 评论

#### POST /api/posts/:postId/comments
发表评论，需登录。

**Body**：
```json
{
  "content": "评论内容..."
}
```

**校验**：content 长度 1-2000。

#### DELETE /api/comments/:id
删评论。只能删自己的评论，管理员可删任意评论。

---

### 5.5 点赞

#### POST /api/posts/:id/like
点赞（toggle 逻辑）。已点赞则取消，未点赞则点赞。需登录。

**响应**：
```json
{
  "liked": true,
  "like_count": 6
}
```
- `liked: true` 表示操作后为已点赞状态
- `like_count` 为操作后帖子的最新点赞数

---

### 5.6 管理端 API（均需 admin 角色）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/users?page=1&pageSize=20 | 用户列表 |
| PUT | /api/admin/users/:id/ban | 封禁 / 解封用户 |
| POST | /api/admin/boards | 创建板块 |
| PUT | /api/admin/boards/:id | 编辑板块（名称 + 描述） |
| DELETE | /api/admin/boards/:id | 删除板块（级联删除板块下所有帖子） |
| DELETE | /api/admin/posts/:id | 删除任意帖子 |
| DELETE | /api/admin/comments/:id | 删除任意评论 |

**创建 / 编辑板块 Body**：
```json
{
  "name": "板块名",
  "description": "板块描述"
}
```

**封禁用户 Body**：
```json
{
  "is_banned": true
}
```

## 六、前端路由

### 6.1 用户端

| 路径 | 页面组件 | 说明 |
|------|----------|------|
| /login | LoginView | 登录页，已登录重定向到 /boards |
| /register | RegisterView | 注册页，已登录重定向到 /boards |
| /boards | BoardListView | 板块列表（首页） |
| /board/:boardId | PostListView | 帖子列表，含分页器 |
| /post/:postId | PostDetailView | 帖子详情 + 评论区 + 评论输入框 + 点赞按钮 |

### 6.2 管理端

| 路径 | 页面组件 | 说明 |
|------|----------|------|
| /login | AdminLoginView | 管理员登录页 |
| /boards | AdminBoardView | 板块管理（列表 + 新增 + 编辑 + 删除） |
| /posts | AdminPostView | 帖子管理（列表 + 删除） |
| /comments | AdminCommentView | 评论管理（列表 + 删除） |
| /users | AdminUserView | 用户管理（列表 + 封禁/解封） |

### 6.3 路由守卫

- **用户端**：登录页/注册页对已登录用户重定向；发帖、评论、点赞等操作在 UI 层判断登录状态，未登录弹出登录提示。
- **管理端**：除 `/login` 外所有页面需 admin 角色，否则重定向到 `/login`。

## 八、开发顺序

按照「后端先行、前端跟进、分层验证」的策略：

| 步骤 | 内容 | 验证方式 |
|------|------|----------|
| 1 | 初始化 NestJS 项目，配置 TypeORM + PostgreSQL 连接 | 启动服务，数据库自动建表 |
| 2 | 定义 User / Board / Post / Comment / Like 实体 | 检查数据库表结构 |
| 3 | 认证模块：注册 + 登录 + JWT 签发 | Postman 调接口验证 |
| 4 | 板块 CRUD（公开接口 + 管理端接口） | Postman 调接口验证 |
| 5 | 帖子 CRUD + 分页 + 权限控制 | Postman 调接口验证 |
| 6 | 评论功能 + 权限控制 | Postman 调接口验证 |
| 7 | 点赞功能（toggle + 计数同步） | Postman 调接口验证 |
| 8 | 管理端用户管理 API（列表 + 封禁） | Postman 调接口验证 |
| 9 | 用户端 Vue 页面开发 | 浏览器逐页面走通 |
| 10 | 管理端 Vue 页面开发 | 浏览器逐页面走通 |
| 11 | 两端联调 + 边界情况测试 | 完整流程走通 |

## 九、待确认事项

以下问题在进入开发前建议明确：

1. **帖子编辑**：是否需要？当前设计只含删不含改，如需编辑可后续追加。
2. **评论回复**：当前评论为单层平铺，是否需要嵌套回复（评论的评论）？
3. **帖子排序**：列表页默认按时间倒序还是按热度（点赞数）排序？
4. **图片上传**：帖子和评论是否支持上传图片？
