# YY论坛项目设计文档

## 一、项目概述

YY论坛是一个轻量图片论坛系统，包含用户端、管理端和后端服务。当前功能已从传统论坛扩展到“个人中心 + 频道导航 + 信息流发现页 + 图片帖详情”的方向，用户端视觉参考图片社区：左侧主导航、顶部搜索、频道栏、瀑布流内容卡片、左图右文详情页。

核心流程：

- 游客：进入推荐页 → 浏览频道/板块 → 查看帖子详情和评论。
- 用户：注册/登录 → 发布图片帖 → 评论 → 点赞 → 管理个人资料和历史帖子。
- 管理员：登录管理端 → 管理板块、用户、帖子、评论。

## 三、产品与界面方向

### 3.1 用户端新视觉

用户端已重构为 YY论坛 图片发现页：

- 左侧固定导航：发现、直播、发布、通知、我、更多。
- 顶部栏：左侧品牌，中间搜索框，右侧登录/用户头像等轻量入口。
- 主内容：多列瀑布流帖子卡片，强调图片、标题、作者头像、昵称、点赞数。
- 顶部频道栏：第一项固定 `推荐`，后续频道来自数据库板块。
- `/boards` 定义为推荐页，不再只是传统板块列表。
- `/board/:boardId` 仍是单板块帖子流。
- 帖子详情页采用左图右文结构：左侧为大图/多图轮播，右侧为作者信息、关注按钮、标题、正文说明、标签、评论列表、底部评论和互动栏。
- 第一版只允许发布图片帖：必须至少上传 1 张图片；文字只作为标题、说明和标签，不支持纯文本长帖。

### 3.2 频道规则

- `推荐` 固定前端写死，不入 `board` 表。
- 管理员每创建一个板块，用户端顶部频道栏自动新增一个对应入口。
- 板块显示顺序由 `board.sort_order` 控制，新建板块默认追加到末尾。
- 当前第一版不做拖拽排序；需要时后续新增排序接口和管理端 UI。

### 3.3 视觉关键词

- 白底、胶囊按钮、轻量阴影、大圆角图片卡片。
- 内容优先，导航安静，不做传统厚重论坛卡片。
- 移动端频道栏横向滚动，卡片流单列或双列自适应。
- 详情页左侧图片占主视觉，右侧内容区保持白底、细分割线、紧凑评论流。

## 五、数据库设计

### 5.1 实体关系

```text
User 1 ── N Post N ── 1 Board
User 1 ── N Comment N ── 1 Post
User 1 ── N PostLike N ── 1 Post
Post 1 ── N PostImage
```

### 5.2 User

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | 用户名，唯一 |
| email | VARCHAR(100) | 邮箱，唯一 |
| password | VARCHAR(255) | bcrypt 哈希 |
| role | ENUM('user','admin') | 角色 |
| is_banned | TINYINT(1) | 是否封禁 |
| avatar_url | VARCHAR(255) | 头像地址 |
| signature | VARCHAR(200) | 个性签名 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.3 Board

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) | 板块名，唯一 |
| description | VARCHAR(200) | 板块描述 |
| sort_order | INT | 频道排序 |
| created_at | DATETIME | 创建时间 |

### 5.4 Post

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| title | VARCHAR(200) | 标题 |
| content | TEXT | 正文 |
| user_id | INT | 发帖人 |
| board_id | INT | 所属板块 |
| like_count | INT | 点赞数冗余计数 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5.5 PostImage

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| post_id | INT | 所属帖子 |
| url | VARCHAR(255) | 图片地址 |
| sort_order | INT | 图片排序 |
| created_at | DATETIME | 创建时间 |

### 5.6 Comment

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| content | TEXT | 评论内容 |
| user_id | INT | 评论人 |
| post_id | INT | 所属帖子 |
| created_at | DATETIME | 创建时间 |

### 5.7 PostLike

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| user_id | INT | 点赞人 |
| post_id | INT | 被赞帖子 |
| created_at | DATETIME | 创建时间 |

`post_like` 使用 `UNIQUE(user_id, post_id)` 防重复点赞，避免 MySQL 关键字 `like` 冲突。

## 六、API 设计

### 6.1 通用约定

- 基础路径：`/api`
- 认证：`Authorization: Bearer <token>`
- 分页：`?page=1&pageSize=10`
- 分页响应：`{ items, total, page, pageSize }`
- 详细接口状态以 `docs/api-client.csv` 和 `docs/api-admin.csv` 为准。

### 6.2 用户端核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 注册 |
| POST | `/api/auth/login` | 登录 |
| GET | `/api/boards` | 板块/频道列表，返回 `sort_order` |
| GET | `/api/boards/:id` | 板块详情 |
| GET | `/api/boards/:boardId/posts` | 板块帖子分页 |
| POST | `/api/boards/:boardId/posts` | 发帖 |
| POST | `/api/posts/:id/images` | 上传帖子图片 |
| DELETE | `/api/posts/:postId/images/:imageId` | 删除帖子图片 |
| GET | `/api/feed` | 推荐/频道信息流，返回封面图和图片列表 |
| GET | `/api/posts/search` | 搜索帖子 |
| GET | `/api/posts/:id` | 帖子详情，含图片、评论和头像 |
| PUT | `/api/posts/:id` | 编辑帖子 |
| DELETE | `/api/posts/:id` | 删除帖子 |
| POST | `/api/posts/:postId/comments` | 发表评论 |
| DELETE | `/api/comments/:id` | 删除评论 |
| POST | `/api/posts/:id/like` | 点赞/取消点赞 |
| GET | `/api/me` | 我的资料 |
| PUT | `/api/me/profile` | 修改签名 |
| POST | `/api/me/avatar` | 上传头像 |
| PUT | `/api/me/password` | 修改密码 |
| GET | `/api/me/posts` | 我的帖子 |

### 6.3 管理端核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/users` | 用户列表 |
| PUT | `/api/admin/users/:id/ban` | 封禁/解封用户 |
| POST | `/api/admin/boards` | 创建板块，自动生成 `sort_order` |
| PUT | `/api/admin/boards/:id` | 编辑板块 |
| DELETE | `/api/admin/boards/:id` | 删除板块 |
| DELETE | `/api/admin/posts/:id` | 删除任意帖子 |
| DELETE | `/api/admin/comments/:id` | 删除任意评论 |

## 九、重构计划

用户端按 YY论坛 图片信息流界面继续打磨：

1. 布局层：新增左侧固定导航、顶部搜索、右侧操作入口。
2. 推荐页：`/boards` 改为瀑布流信息流，保留频道栏第一项 `推荐`。
3. 帖子卡片：展示真实图片封面、标题、作者头像昵称、点赞数。
4. 帖子详情：重构为左图右文，小屏改为上图下内容。
5. 发帖：改为图片必填，至少 1 张，最多 9 张；标题/说明保留。
6. 数据层：已新增 `post_image`，前端需要接入上传和展示。

## 十、当前约束

- 本次先确认帖子详情和图片帖方向，不直接重构界面代码。
- 纯文本帖子不再作为新发帖目标；旧纯文本帖子可保留展示，但新发布流程要要求图片。
- 若旧帖子没有图片，详情页左侧显示占位或兼容文字卡，避免白屏。
- 管理员创建板块已能驱动用户端频道栏自动追加。
